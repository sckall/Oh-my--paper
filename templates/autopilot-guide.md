# 🚀 自动驾驶指南

## 最简单的用法

```
/omp:mega
→ 选择 "🚀 自动驾驶"
→ 然后就不用管了！
```

## 你可以说的关键词

| 你说 | 效果 |
|------|------|
| `开始` 或 `自动驾驶` | 启动全自动流水线 |
| `继续` | 从暂停处恢复 |
| `暂停` | 停在当前阶段 |
| `状态` | 显示当前进度 |
| `停止` | 完全停止 |

## 自动驾驶流程

```
A1 → A+ → A2 → A3 → A4 → A5 → A6 → A7 → A8
    ↓
B9 → B10 → B11 → B12 → B13 → B14 → B15
    ↓
C16 → C17 → C18 → C19 → C20 → C21 → C22 → C23
    ↓
D24 → I25
```

## 什么时候会暂停？

自动驾驶会在这些决策点暂停，让你做重要决定：

| 决策点 | 原因 | 你可以 |
|--------|------|--------|
| A8/B15 | REFINE | 调整或继续 |
| A8/B15 | PIVOT | 确认重大变更 |
| A5/B10/C20 | 门控未通过 | 修改或跳过 |
| - | CRITICAL 问题 | 查看并决定 |

## 暂停时你可以说

```
[查看详情]  — 看详细原因
[接受建议]  — 按系统建议继续
[手动决策]  — 你来决定下一步
[继续]      — 从暂停处恢复
```

## 进阶用法

### 半自动模式

```
/omp:mega
→ 选择 "📋 半自动"
→ 每阶段前会确认
```

### 单步执行

```
/omp:mega
→ 选择 "▶️ 单步"
→ 执行一个阶段后停止
```

### 查看所有命令

```
/omp:help
```

## 设置

在 `.pipeline/memory/settings.md` 中配置：

```markdown
AUTO_EXECUTE: true     # 启用自动驾驶
AUTO_DECISION: false   # 决策需确认
PAUSE_ON_PIVOT: true   # PIVOT 时暂停
```
