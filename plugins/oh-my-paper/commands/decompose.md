> **必须使用 AskUserQuestion 工具进行所有确认步骤，不得用纯文字替代。**

你是 Oh My Paper Orchestrator。此命令实现 A2/A3 问题分解，将研究主题分解为结构化问题树。

## 第一步：读取研究主题

```bash
cat .pipeline/memory/project_truth.md
cat .pipeline/docs/research_brief.json
```

用 `AskUserQuestion` 确认研究主题：

> **问题分解**
>
> 研究主题：[主题]
>
> 目标：将此主题分解为可研究的具体问题
>
> 选项：
> - `确认，开始分解`
> - `先调整研究主题`
> - `取消`

## 第二步：执行问题分解

调用 LLM 将研究主题分解为结构化问题树：

```
/codex:rescue 使用 problem_decompose 技能，将研究主题分解为至少 4 个优先级的子问题。

研究主题：[主题]

输出格式：
```markdown
## 问题来源
[主题如何来源，现有研究背景]

## 子问题列表

### Q1: [子问题名称] (最高优先级)
- **问题描述**: [具体问题]
- **研究价值**: [为什么重要]
- **可行性**: [高/中/低]
- **风险**: [主要风险]

### Q2: [子问题名称]
...

### Q3: ...
### Q4: ...
```

将结果写入 .pipeline/docs/problem_tree.md
```

## 第三步：展示问题树

读取问题树，用 `AskUserQuestion` 展示：

> **问题分解结果**
>
> | 优先级 | 问题 | 可行性 | 主要风险 |
> |-------|------|--------|---------|
> | Q1 | [问题1] | [高/中/低] | [风险1] |
> | Q2 | [问题2] | [中/低] | [风险2] |
> | Q3 | [问题3] | ... | ... |
> | Q4 | [问题4] | ... | ... |
>
> 选项：
> - `确认，继续`
> - `调整问题优先级`
> - `添加/删除问题`
> - `取消`

## 第四步：生成搜索策略

基于问题树，生成文献搜索策略：

```
/codex:rescue 基于问题树生成搜索策略，覆盖：
- 核心主题
- 相关方法
- 数据集/基准
- 理论基础

每个策略 3-5 个关键词查询，共至少 8 个查询。

输出格式：
```yaml
search_strategies:
  - name: 核心主题
    queries:
      - "[关键词1]"
      - "[关键词2]"
  ...
sources:
  - id: openalex
    name: OpenAlex
    type: API
    url: https://api.openalex.org
```

将结果写入 .pipeline/docs/search_strategy.yaml
```

## 第五步：更新项目状态

更新 `project_truth.md`：

```markdown
## 问题分解

完成时间：[时间戳]

### 问题树
[问题树摘要]

### 搜索策略
[策略摘要]
```

## 输出文件

| 文件 | 内容 |
|------|------|
| `.pipeline/docs/problem_tree.md` | 完整问题树 |
| `.pipeline/docs/search_strategy.yaml` | 搜索策略 |

## 问题分解结构

```
研究主题
    │
    ├── Q1: [核心问题] (最高优先级)
    │       ├── 子问题 1.1
    │       └── 子问题 1.2
    │
    ├── Q2: [相关问题]
    │       ├── 子问题 2.1
    │       └── 子问题 2.2
    │
    ├── Q3: [理论基础]
    │       └── ...
    │
    └── Q4: [扩展应用]
            └── ...
```

## 优先级排序标准

| 优先级 | 标准 |
|-------|------|
| 最高 | 问题核心、可行性强、风险低 |
| 高 | 问题重要、可行性中、风险可控 |
| 中 | 问题相关、可行性低、需要更多资源 |
| 低 | 问题边缘、可选探索 |
