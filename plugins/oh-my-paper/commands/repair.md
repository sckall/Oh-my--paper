## 核心设计理念

**自修复原则**：
- 不使用 try-except 掩盖错误
- 不使用 np.nan_to_num() 粘合伤口
- **追踪根源**，修复代码逻辑

## 自动执行流程

### 1. 检测问题

```bash

cat .pipeline/mega/logs/experiment_log.md

# 查找 NaN/Inf
grep -E "NaN|Inf|Warning|RuntimeWarning" .pipeline/mega/logs/experiment_log.md
```

调用修复分析：

```

可能原因：

- 初始化问题

### 3. 执行修复

根据诊断结果，修复代码：

**常见修复模式**：

| 问题 | 修复方案 |
|------|---------|
| 学习率过高 | 降低学习率，添加梯度裁剪 |
| 零除错误 | 添加 eps 或检查分母 |
| 未归一化 | 添加归一化层 |
| 数值溢出 | 缩放输入，添加稳定性检查 |
