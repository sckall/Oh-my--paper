# 结论与反思 写作指南

## 1. 核心任务

本章（第5章）需总结全文并升华反思，需包含：
- 研究工作的主要贡献
- 开发过程中的关键经验
- 研究的不足与未来方向
- 对一线教师利用 AI 开发教学资源的建议

## 2. 内容要素

### 2.1 主要贡献

用 3-4 个要点概括本文的核心价值，每个要点一句话：
1. （功能贡献）程序支持5种遗传模式的系谱图绘制与模拟，填补了现有教学资源在遗传规律探究方面的不足
2. （方法贡献）系统展示了 AI 辅助开发虚拟实验的完整迭代流程，为教师提供了可复用的开发范式
3. （教学贡献）通过教学实践验证了虚拟实验在提升学生遗传推理能力方面的有效性
4. （资源贡献）程序可作为开源资源，供一线教师直接使用或二次开发

### 2.2 经验与反思

分两个维度展开：

**AI 辅助开发的经验**
- 如何设计有效的提示词（教训：不要一次性要求太多功能）
- 迭代优化的节奏（教训：每轮聚焦一个问题）
- 教师角色定位（教训：AI 生成代码需教师把关，不能完全依赖）

**虚拟实验教学的经验**
- 虚拟实验的适用边界（可模拟推理过程，不能替代真实实验技能）
- 双模式设计（验证 vs 探究）的教学价值
- 电泳图模块在分子层面理解中的辅助作用

### 2.3 不足与未来方向

客观陈述局限性，并指明改进方向：
1. **不足**：目前仅支持单基因遗传病模拟，未涉及多基因遗传、基因组印记等复杂情况
2. **不足**：教学实践的样本量较小（仅在2个班级实施），效果的外推性有待验证
3. **未来方向**：增加多基因遗传模拟、基因互作等拓展模块
4. **未来方向**：开展更大规模的教学实验，积累实证数据
5. **未来方向**：开发移动端版本，降低设备门槛

### 2.4 对一线教师的建议

用 2-3 段话为有意使用 AI 开发教学资源的教师提供可操作的建议：
- 工具选择（Google AI Studio / 通义灵码 / Claude 等）
- 提示词撰写的基本原则
- 迭代优化的心态准备

## 3. 结构规范

建议结构：
```
5 结论与反思
  5.1 主要贡献
  5.2 经验与反思
  5.3 不足与未来方向
  5.4 对一线教师的建议
```

## 4. 常见问题

| 问题 | 解决 |
|------|------|
| 结论写成摘要 | 摘要回答"做了什么"，结论回答"意味着什么" |
| 反思过于 superficial | 需包含具体失败案例和改进过程 |
| 未来方向与正文脱节 | 每个方向都能在正文找到依据（"由于篇幅限制，未能..."） |

## 5. 参考模板

参考期刊模板.md的总体格式。结论部分篇幅建议 600-800 字，反思部分 400-600 字。
