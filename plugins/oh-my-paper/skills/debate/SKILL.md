## 自动执行流程

### 1. 读取假设（自动）

```bash

```

### 2. Round 1: Proponent 发言（自动）

```

假设：[假设内容]

任务：支持假设，准备最强论点。

输出格式：
## Proponent 论点

### 防御准备
```

写入 `.pipeline/parallel/debate/round1_proponent.md`

### 3. Round 1: Opponent 反驳（自动）

```
你是假设挑战者（Opponent）。

假设：[假设内容]
正方论点：[读取 round1_proponent.md]

任务：挑战假设，找出漏洞。

输出格式：
## Opponent 论点
### 最大漏洞 (1-3)
### 反例/矛盾
### 攻击正方
```

写入 `.pipeline/parallel/debate/round1_opponent.md`

### 4. Round 2: Synthesizer 裁决（自动）

```
你是中立综合者（Synthesizer）。

假设：[假设内容]
正方论点：[读取 round1_proponent.md]
反方论点：[读取 round1_opponent.md]

任务：权衡双方，给出裁决。

输出格式：
## Synthesizer 裁决
### 论点强度评估
| 维度 | 正方 | 反方 |
|------|------|------|
### 最终裁决：通过/修改后通过/否决
### 理由
### 建议
```

写入 `.pipeline/parallel/debate/round2_synthesis.md`

**裁决说明**：

|------|------|---------|
| 通过 | 正方 > 30/40 | 进入 A8 决策 |

| 否决 | 反方 ≥ 正方 | 返回 /omp:ideate |

## 裁决标准

| 分数 | 含义 |
|------|------|
| 9-10 | 无可争议的强论点 |
| 7-8 | 有实质支持，但有微小瑕疵 |
| 5-6 | 论点存在，但有明显漏洞 |
| 3-4 | 论点薄弱，容易被攻击 |
| 1-2 | 论点站不住脚 |
