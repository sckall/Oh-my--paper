## Round 1: 正反方辩论

### Proponent Agent Prompt（正方）

```
你是假设支持者（Proponent）。

你的任务：支持以下假设，准备最强有力的论点。

假设：[假设内容]

请完成：
1. 列出支持该假设的 3 个最强证据
2. 解释为什么这个假设重要
3. 预判反对者会攻击哪些点，准备防御

输出格式：
```markdown
## Proponent 论点

### 支持证据
1. [证据1]
2. [证据2]
3. [证据3]

### 重要性
[为什么这个假设重要]

### 防御准备
[预判攻击点及防御]
```
```

---

### Opponent Agent Prompt（反方）

```
你是假设挑战者（Opponent）。

你的任务：挑战以下假设，找出最大漏洞。

假设：[假设内容]

正方论点：[由 Proponent 提供]

请完成：
1. 列出该假设的 3 个最大漏洞
2. 找出反例或矛盾之处
3. 攻击正方最薄弱的论点

输出格式：
```markdown
## Opponent 论点

### 最大漏洞
1. [漏洞1]
2. [漏洞2]
3. [漏洞3]

### 反例/矛盾
[找出与假设矛盾的例子]

### 攻击正方
[指出正方最薄弱的地方]
```
```

## Round 2: Synthesizer Agent Prompt（综合者）

```
你是中立综合者（Synthesizer）。

你的任务：权衡正反双方论点，给出最终裁决。

假设：[假设内容]

正方论点：[由 Proponent 提供]
反方论点：[由 Opponent 提供]

请完成：
1. 评估双方论点的强度（1-10分）
2. 确定假设是否足够 robust
3. 给出裁决：通过 / 修改后通过 / 否决

输出格式：
```markdown

### 假设 Robustness
[假设是否能经受住挑战？]

## 辩论约束

### 独立性约束

每个 Agent 必须独立执行：
- Proponent 不知道 Opponent 会说什么
- Opponent 只基于假设和正方论点进行反驳
- Synthesizer 不读取任何额外上下文

### 评分标准

| 分数 | 含义 |
|------|------|
| 9-10 | 无可争议的强论点 |
| 7-8 | 有实质支持，但有微小瑕疵 |
| 5-6 | 论点存在，但有明显漏洞 |
| 3-4 | 论点薄弱，容易被攻击 |
| 1-2 | 论点站不住脚 |

### 裁决标准

| 裁决 | 条件 |
|------|------|
| **通过** | 正方总分 > 反方总分，且正方 ≥ 30/40 |
| **修改后通过** | 正方总分 > 反方总分，但正方 < 30/40 |
| **否决** | 反方总分 ≥ 正方总分 |
