# 快速检查清单

在交付文本前，进行以下检查：

- 连续三个句子长度相同？打断其中一个。
- 段落以简洁的单行结尾？变换结尾方式。
- 揭示前有破折号？删除它。
- 解释隐喻或比喻？相信读者能理解。
- 使用了“此外”“然而”等连接词？考虑删除。
- 三段式列举？改为两项或四项。
- 出现“修改后”“改写后”“可以改成”“如果你愿意”等元话语？必须删除。
- 看起来像聊天回复、润色建议、答疑说明或候选标题列表？必须改回正文。
- 原文核心意思、事实、论点、结论或因果关系发生变化？必须回退并重写。
- 标题、编号、章节层级、段落顺序被改动？必须恢复。

## 处理流程

1. 仔细阅读输入文本。
2. 识别上述所有模式的实例。
3. 重写每个有问题的部分。
4. 确保修订后的文本：
   - 大声朗读时听起来自然
   - 自然地改变句子结构
   - 使用具体细节而不是模糊的主张
   - 为上下文保持适当的语气
   - 适当时使用简单的结构（是/有）
   - 仍然保持原文核心意思、事实和结论不变
   - 只呈现正文，不含说明性前后缀或建议性文字
5. 呈现人性化版本。

## 输出格式

提供：

1. 纯净的重写后正文

不要附加总结、说明、注解、候选版本或邀请式补充语。

## 质量评分

对改写后的文本进行 1-10 分评估，总分 70。

| 维度     | 评估标准                                                                  | 得分 |
| -------- | ------------------------------------------------------------------------- | ---- |
| 直接性   | 直接陈述事实还是绕圈宣告？10 分：直截了当；1 分：充满铺垫                 | /10  |
| 节奏     | 句子长度是否变化？10 分：长短交错；1 分：机械重复                         | /10  |
| 信任度   | 是否尊重读者智慧？10 分：简洁明了；1 分：过度解释                         | /10  |
| 真实性   | 听起来像真人说话吗？10 分：自然流畅；1 分：机械生硬                       | /10  |
| 精炼度   | 还有可删减的内容吗？10 分：无冗余；1 分：大量废话                         | /10  |
| 语义保真 | 原文意思、事实、论点、结论是否完整保留？10 分：完全保留；1 分：明显漂移   | /10  |
| 纯净输出 | 是否彻底避免说明、建议、候选版本、聊天腔？10 分：完全纯净；1 分：污染明显 | /10  |
| 总分     |                                                                           | /70  |

评分标准：

- 63-70 分：优秀，已去除明显 AI 痕迹且语义稳定。
- 49-62 分：良好，仍有改进空间。
- 低于 49 分：需要重新修订。

## 使用说明

本文件既可用于两轮改写过程中的阶段性评分，也用于最终终稿的检查和评分，不替代两轮主提示词本身。

推荐使用方式：

- 在第 1、2 轮结束后，基于当前轮的文本做一次快速评分，可以只简要给出各维度得分和总分，用于观察两轮之间的改写效果变化和问题聚焦；
- 在完整执行 `baibaiAIGC1.md` -> `baibaiAIGC2.md` 两轮之后，对“最终改写结果”再进行一轮全面检查与评分，此时应输出完整评分表，并据此决定是否需要进一步修订终稿。

两轮阶段性评分和最终终检评分要分开记录，最终交付时以终检评分为主。
