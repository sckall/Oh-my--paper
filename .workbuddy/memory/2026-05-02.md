# 工作日志 - 2026-05-02

## 实现 OMP 插件更新功能

### 新增功能
1. **`/omp:update` 命令** - 一键更新插件到最新版本
   - 支持 `--check-only` 参数（仅检查更新）
   - 支持 `--auto` 参数（自动更新，不询问确认）
   - 从 GitHub 自动下载最新代码并安装到插件缓存目录

2. **自动更新检查机制**
   - 在 SessionStart hook 中增加检查逻辑（每日最多一次）
   - 发现新版本时提示用户运行 `/omp:update`
   - 检查记录存储在 `.pipeline/.last-update-check` 文件

### 新增文件
- `skills/omp:update/SKILL.md` - 更新技能文档
- `plugins/oh-my-paper/commands/update.md` - `/omp:update` 命令实现
- `plugins/oh-my-paper/scripts/check-update.mjs` - 版本检查脚本
- `plugins/oh-my-paper/scripts/do-update.mjs` - 更新执行脚本

### 修改文件
- `plugins/oh-my-paper/scripts/on-session-start.mjs` - 添加自动更新检查
- `README.md` - 添加 `/omp:update` 命令说明
- `README.zh.md` - 添加 `/omp:update` 命令说明（中文）

### 修复问题
- 修复脚本路径引用错误（从 `scripts/` 移到 `plugins/oh-my-paper/scripts/`）
- 修复 ES 模块中使用 `require` 的错误
- 修复中英文 README 中重复的"Update"标题

### 使用方式
```bash
# 检查更新
/omp:update --check-only

# 执行更新
/omp:update

# 自动更新（不询问确认）
/omp:update --auto
```

## 三大核心体验改进（Feature 1-3）

### 1. 错误恢复与快照机制
- 新增 `scripts/take-snapshot.mjs`：关键操作前自动打快照（存完整文件内容到 `.pipeline/memory/snapshots/`）
- 新增 `scripts/list-snapshots.mjs`：列出所有可用快照
- 新增 `commands/recover.md`（`/omp:recover` 命令）：从快照恢复项目状态
- 修改 `on-stage-transition.mjs` 和 `on-task-complete.mjs`：插入快照调用
- 每次关键操作前自动打快照，最多保留 20 个

### 2. 进度可视化（改造 `/omp:progress`）
- 重写 `commands/progress.md`：支持 Legacy + Mega 双模式自动检测
- Legacy 模式：ASCII 进度条展示 5 阶段完成度
- Mega 模式：保持原树状图逻辑不变

### 3. 多项目并行支持
- 新增 `commands/switch.md`（`/omp:switch` 命令）：切换活跃项目
- 新增 `commands/new-project.md`（`/omp:new-project` 命令）：创建新子项目
- 修改 `on-session-start.mjs`：读取 `.omp.yml` 显示当前项目名
- `.omp.yml` 管理多项目配置，每个子项目有独立 `.pipeline/`
