\section{Introduction}
Academic writing workflows often fragment across editors, AI chat tools, PDF viewers, and figure-generation utilities. This fragmentation creates friction at exactly the points where iteration should be fastest.

\subsection{Motivation}
Researchers need a workspace where compile feedback, revision tools, and figure ideation remain attached to the same project context.

\subsection{Goal}
The goal of ViewerLeaf is to collapse these context switches while preserving explicit user control over patches and generated figures.
